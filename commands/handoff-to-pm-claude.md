# /handoff-to-pm-claude - 完了報告（PM Claudeへ貼り付け）

実装役 Claude Code が、PM役 Claude Code へ貼り付けるための **完了報告** を生成します。  
Cursor を使わない「ソロ 2-Claude 運用（PM Claude + Impl Claude）」のためのハンドオフです。

## バイブコーダー向け（こう言えばOK）

- 「**PMの自分に完了報告を書いて**」→ このコマンド
- 「**確認観点（何を見ればいいか）も入れて**」→ 受け入れ基準/リスク/確認手順を厚めに書きます
- 「**次の一手も提案して**」→ PMが選べる形で 1〜3案を出します

## できること（成果物）

- PM Claude に渡す **1枚の報告**（概要/変更点/確認/リスク/次アクション）
- `Plans.md` のステータス更新（`cc:完了` / `pm:確認済`）の指示を明確化

## 手順（このコマンドがやること）

1. `git status -sb` / `git diff --name-only` を前提に変更範囲を把握
2. `Plans.md` の対象タスクが **`cc:完了`** になっているか確認（未更新なら更新する提案を含める）
3. 下のフォーマットで PM Claude に貼れる報告を出す

## 出力フォーマット（そのまま PM Claude へ貼る）

```markdown
## 完了報告（Impl Claude → PM Claude）

### 概要（1〜3行）
- （何を達成したか）

### 変更点（要点）
- （ユーザーに見える変更）
- （内部の変更：必要なら）

### 変更ファイル
- （ファイル一覧）

### 動作確認 / テスト
- （実施した確認コマンド）
- （結果）

### 受け入れ基準（満たしたか）
- [ ] （基準1）→ ✅/⚠️ と理由
- [ ] （基準2）

### リスク / 注意点
- （あれば。データ移行・破壊的変更・後方互換など）

### Plans / 進捗（更新指示）
- （このタスクを `cc:完了` にする / 既にしてある）
- （PMレビュー後に `pm:確認済` にする対象）

### 次のアクション（PMが選べる 1〜3案）
1. （案1）
2. （案2）
3. （案3）
```
