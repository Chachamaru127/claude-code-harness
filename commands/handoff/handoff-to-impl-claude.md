# /handoff-to-impl-claude - タスク依頼（Impl Claudeへ貼り付け）

PM役 Claude Code が、実装役 Claude Code（Impl Claude）へ貼り付けるための **依頼文** を生成します。  
Cursor を使わない「ソロ 2-Claude 運用（PM Claude + Impl Claude）」のためのハンドオフです。

## バイブコーダー向け（こう言えばOK）

- 「**このタスクを実装役Claudeに渡したい**」→ このコマンド
- 「**受け入れ基準も付けて**」→ 実装がブレないように明文化します
- 「**安全に進めて**」→ 触って良い範囲/禁止事項/検証方法まで含めます

## できること（成果物）

- Impl Claude に渡す **コピー&ペースト用の依頼文**
- `Plans.md` のタスクに **`pm:依頼中`** を付ける運用を踏襲（互換: `cursor:依頼中`）

## 使い方

```text
/handoff-to-impl-claude
```

または自然言語で：
- 「実装役にこのタスクを渡して」

## ✅ まず最初に（忘れ防止チェックリスト）

- [ ] `Plans.md` にタスクを追加/更新し、**`pm:依頼中`** を付与（互換: `cursor:依頼中`）
- [ ] 受け入れ基準（最低2つ）を `Plans.md` 側にも残す（後で迷子にならない）
- [ ] その後に、このコマンドで Impl Claude に渡す依頼文を生成する

## 出力フォーマット（そのまま Impl Claude へ貼る）

```markdown
## タスク依頼（PM Claude → Impl Claude）

### 背景 / 目的
- （なぜやるか）

### 期待する成果（ユーザーに見える形）
- （何ができるようになればOKか）

### 受け入れ基準（必須）
- [ ] （基準1）
- [ ] （基準2）

### スコープ（やる / やらない）
- **やる**:
  - （項目）
- **やらない**:
  - （項目）

### 変更対象（目安）
- （ファイル/ディレクトリ/機能）

### 制約 / 禁止事項
- （例: 破壊的変更禁止、秘密情報をコミットしない、など）

### 進め方（推奨）
1. 現状把握（`git status -sb` / `Plans.md` / 関連コード）
2. 小さく実装 → 動作確認 → テスト
3. `Plans.md` を `cc:WIP` → `cc:完了` まで更新
4. 完了したら `/handoff-to-pm-claude` で完了報告を作って返す

### Plans（マーカー運用）
- この依頼タスクは `pm:依頼中`（互換: `cursor:依頼中`）
- 着手したら `cc:WIP`
- 完了したら `cc:完了`
- PMレビュー後に `pm:確認済`（互換: `cursor:確認済`）
```
