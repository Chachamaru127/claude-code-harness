# タスク: impl-test

## 概要
既存コードに対するテストファイルを作成する。

## テスト対象機能
- `/work` コマンド
- テスト生成能力
- 既存コードの理解

## 前提条件
`src/utils/string-helpers.ts` が存在すること（impl-utility タスクの後に実行）

## プロンプト

```
src/utils/string-helpers.ts のテストを作成してください。

テストフレームワーク: Jest
出力先: src/utils/__tests__/string-helpers.test.ts

以下を含めてください:
- 正常系テスト
- エッジケース（空文字、長い文字列）
- 型チェックが効いていることの確認
```

## 期待される出力
- `src/utils/__tests__/string-helpers.test.ts` が作成される
- 各関数に対するテストがある
- エッジケースが考慮されている

## 成功基準
| 基準 | 条件 |
|------|------|
| ファイル作成 | テストファイルが存在 |
| テスト数 | 6個以上（各関数×2ケース以上） |
| カバレッジ | 主要パスをカバー |
| 実行可能 | jest で実行できる |

## 測定ポイント
- テスト作成までの時間
- 生成されたテストケース数
- ファイル読み取り回数（既存コード理解）
